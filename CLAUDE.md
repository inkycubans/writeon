This file provides context for AI assistants working in this repository.

## Project Overview

**WriteOn** is a Swedish-language AI-assisted writing tool for novelists, screenwriters, and playwrights. It is a fully client-side single-page application with no backend database — all data lives in the user's browser via `localStorage`.

- **Live URL**: https://writeon-production-d435.up.railway.app
- **Platform**: Railway.app (auto-deploys from `master` branch on GitHub)
- **Language**: Swedish (UI text, documentation, and variable names are in Swedish)

---

## Tech Stack

| Layer        | Technology                              |
|--------------|-----------------------------------------|
| Frontend     | Vanilla JavaScript + HTML + CSS (monolithic `index.html`) |
| Styling      | Custom CSS with CSS variables; Tailwind CSS available but used minimally |
| Build Tool   | Vite                                    |
| Server       | Express.js (`server.js`) — static file serving only |
| AI API       | Google Gemini (`gemini-2.5-flash-lite`) via direct browser fetch |
| Export libs  | html2canvas, jsPDF, docx.js, jszip (loaded from CDN) |
| Icons        | lucide-react (available but CDN icons also used) |
| Persistence  | Browser `localStorage` only            |

> **Note**: `package.json` lists React as a dependency and there is a `src/` directory with React components, but the **live application is entirely in `index.html`** using vanilla JavaScript. The `src/` React code is not currently used.

## Architecture: The Monolith

The application lives almost entirely in `/index.html`. Do not look for business logic in `src/` — it is not used.

`index.html` is structured as:
1. `<head>` — CDN imports (jsPDF, html2canvas, docx.js, jszip), favicon
2. `<style>` — ~3500 lines of custom CSS with CSS variables
3. `<body>` — Full HTML structure for all views and panels
4. `<script>` — ~1500 lines of vanilla JavaScript

### Views / Sections

The app uses a view-switching pattern. Call `switchView(viewName)` to navigate:

| View ID        | Description                        |
|----------------|------------------------------------|
| `editor`       | Main manuscript editor             |
| `coach`        | AI literary coach chat             |
| `ideabank`     | Genre, synopsis, idea tracking     |
| `characters`   | Character builder + AI chat        |
| `worldbuild`   | Drag-and-drop world-building cards |
| `research`     | Categorized research notes         |

### The `COACHES` Object

Six AI literary coaches are defined as a global constant:

```javascript
const COACHES = {
  king:      { name, initial, colors, desc, placeholder, greeting, systemPrompt },
  lamott:    { ... },   // Anne Lamott
  hemingway: { ... },
  gaiman:    { ... },   // Neil Gaiman
  marquez:   { ... },   // García Márquez
  oates:     { ... },   // Joyce Carol Oates
}
```

### AI API Call Pattern

All Gemini calls follow this shape — direct browser fetch, no backend relay:

```javascript
async function sendCoachMessage() {
  const response = await fetch(
    `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash-lite:generateContent?key=${apiKey}`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        system_instruction: { parts: [{ text: systemPrompt + context }] },
        contents: [...history, { role: 'user', parts: [{ text: input }] }],
        generationConfig: { maxOutputTokens: 600 }
      })
    }
  );
  const data = await response.json();
  return data.candidates[0].content.parts[0].text;
}
```

When no API key is set, the app returns a hardcoded mock response.

### Coach Container Helper

Because multiple coach views can exist in the DOM, always use:

```javascript
function coachContainer() {
  // Returns the currently visible coach UI element
  return document.querySelector('.coach-view-wrap:not([style*="display: none"])');
}
```

Do NOT query `.coach-panel` or `.coach-messages` directly — always go through `coachContainer()`.

### DOM Manipulation

The app uses direct DOM manipulation (no virtual DOM):
- `document.getElementById(id)`
- `element.innerHTML = '...'`
- `element.addEventListener('click', handler)`
- Inline `onclick="functionName()"` attributes in HTML

## Common Tasks for AI Assistants

### Adding a New Literary Coach

1. Add an entry to the `COACHES` constant in `index.html` with: `name`, `initial`, `colors`, `desc`, `placeholder`, `greeting`, `systemPrompt`
2. Add the coach selector button to the coach panel HTML
3. The rest of the system is data-driven

### Modifying AI Behavior

- Edit `systemPrompt` within the relevant coach in `COACHES`
- Adjust `maxOutputTokens` in `generationConfig` if response length needs to change
- The `fix_coach.py` utility script can be used to batch-update coach prompts

### Changing the Gemini Model

Find and replace the model name in the fetch URL:
```
gemini-2.5-flash-lite  →  <new-model-name>
```
This string appears in `sendCoachMessage()`, `sendCharMessage()`, and similar async functions.

### Adding a New View/Section

1. Add a wrapper `<div class="<name>-wrap">` in the HTML body
2. Add a `case '<name>':` branch in `switchView()`
3. Add a nav item to trigger it

### Updating Documentation

Documentation is in Swedish. The four log files (`README.md`, `SNABBSTART.md`, `ARBETSLOGG.md`, `DAGBOK.md`) are for human readers; keep them in Swedish.

## Important Warnings

- **Do not refactor `index.html` into React** without explicit instruction — the monolithic structure is intentional for deployment simplicity.
- **Do not add a backend database** — user data privacy (localStorage-only) is a stated design goal.
- **The API key is never sent to any server** — it goes directly from the browser to Google's API. Do not introduce any backend route that would proxy API key traffic.
- **UI text is in Swedish** — keep all user-facing strings in Swedish unless told otherwise.
- **The `src/` React components are not in use** — do not import from or modify them unless explicitly asked to migrate the app to React.
